## 충돌사례

### case#1

원격레포와 로컬 레포의 커밋 이력이 같은 상황에서 로컬에서
working directory에서의 수정 이후 git pull 한다면

- 에러 발생하지 않음

```bash
# pull = fetch + merge
# fetch는 성공인데 merge가 실패한 경우
git pull origin main
```

충돌이 아니므로 그냥 git push 하면됨

### case#2
git pull https://github.com/piyonoky-maker/pull-conflict.git
  원격에 있는 소스만 받아옴

git push


원격레포에서 commit 이력을 추가했을 때 충돌이 없다.
git pull origin main을 통해 commit이력을 내려 받은 후
병합의 일종으로 볼 수 있으나, 변경사항에 대한 head만
이동하므로 fast forward에 해당함

### case#3
원격에 커밋이력 추가 발생시킨.
또 로컬에서도 별도의 커밋이력을 발생시킴.
단 같은 파일에 대한 수 정이 아닐 때 -> 충돌 발생 안함
git pull 시 분제 없이 실행 되고 단 fase forward가 아닌
머지 커밋이 일어난다.
이 경우 다시 push가 필요하다.
pull = fetch + merge
